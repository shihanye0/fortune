# Spec 018: 每日推送调度

> 模块：推送服务 | 优先级：P0 | 预计：1 天

## 功能描述

定时触发每日运势推送，支持用户自定义推送时间。

## API

```
POST /api/v1/internal/daily-push
```

## 推送流程

```
GitHub Actions 每小时触发
  → 调用 daily-push 接口
  → 查询 push_enabled=TRUE 且 push_time 在当前小时内的用户
  → 对每个用户（并发）：
    1. 读取八字档案
    2. 计算今日运势
    3. 调用 DeepSeek 解读
    4. 推送（QQ 邮箱 / 飞书 / 两者）
    5. 存入数据库
```

## 验收标准

```
Scenario: 定时推送触发
  Given 当前时间 07:00
  And   有 3 个用户设置 push_time="07:00"
  When  GitHub Actions 触发 daily-push
  Then  3 个用户都应收到运势推送
  And   运势应存入数据库

Scenario: 不推送未开启的用户
  Given 用户A push_enabled=FALSE
  When  daily-push 执行
  Then  用户A 不应收到推送

Scenario: DeepSeek API 故障降级
  Given DeepSeek API 不可用
  When  daily-push 执行
  Then  运势数据照常推送
  And   解读文字显示 "今日解读暂不可用"
```

## 技术要点

- GitHub Actions cron: `0 * * * *`（每小时）
- 并发推送：使用 asyncio 并发，控制并发数（避免 API 限流）
- 推送失败重试：3 次，间隔 5 分钟
- 内部接口需验证请求来源（API Key 或签名）
