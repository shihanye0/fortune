# Spec 004: 用户注册

> 模块：用户管理 | 优先级：P0 | 预计：1 天

## 功能描述

用户通过邮箱、密码、生辰信息注册账号，系统自动排算八字并存储。

## API

```
POST /api/v1/auth/register
```

**请求体：**
```json
{
  "username": "张三",
  "email": "zhangsan@qq.com",
  "password": "Abc123456",
  "birth_year": 1990,
  "birth_month": 5,
  "birth_day": 15,
  "birth_hour": 8,
  "gender": 1,
  "birth_location": "北京"
}
```

**响应：**
```json
{
  "success": true,
  "data": {
    "id": 1,
    "username": "张三",
    "email": "zhangsan@qq.com",
    "token": "jwt-token..."
  }
}
```

## 验收标准

```
Scenario: 使用有效信息注册
  Given 邮箱 "zhangsan@qq.com" 未被注册
  When  用户提交完整的注册信息
  Then  应注册成功
  And   应返回 JWT Token
  And   系统应自动排算八字并存入 bazi_profiles 表

Scenario: 邮箱已被注册
  Given 邮箱 "zhangsan@qq.com" 已被注册
  When  用户使用该邮箱注册
  Then  应返回错误 "该邮箱已被注册"
  And   HTTP 状态码应为 409

Scenario: 出生日期无效
  Given 用户在注册页面
  When  用户输入出生日期 "2000-02-30"
  Then  应返回错误 "出生日期无效"

Scenario: 出生日期在未来
  Given 用户在注册页面
  When  用户输入出生日期 "2030-01-01"
  Then  应返回错误 "出生日期不能是未来日期"

Scenario: 密码强度不足
  Given 用户在注册页面
  When  用户输入密码 "123"
  Then  应返回错误 "密码长度至少8位，需包含字母和数字"
```

## 技术要点

- 密码使用 bcrypt 哈希存储
- 注册成功后自动调用八字排盘引擎
- 子时（23:00-01:00）需正确处理日柱切换
- 输入验证：邮箱格式、密码强度、日期有效性
