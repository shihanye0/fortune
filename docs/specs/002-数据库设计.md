# Spec 002: 数据库设计与 ORM

> 模块：基础设施 | 优先级：P0 | 预计：1 天

## 功能描述

创建 MySQL 数据库、编写 SQLAlchemy ORM 模型、配置 Alembic 迁移。

## 数据表

| 表名 | 说明 | 核心字段 |
|------|------|---------|
| users | 用户表 | username, email, password_hash, birth_*, push_*, feishu_webhook |
| bazi_profiles | 八字档案 | user_id, 四柱, day_master, five_elements, ten_gods, major_luck_cycles |
| daily_fortunes | 每日运势 | user_id, date, overall_score, 四维度运势, llm_interpretation, 反馈字段 |
| divination_records | 占卜记录 | user_id, type, question, raw_data, llm_interpretation, 反馈字段 |

## 验收标准

```
Scenario: 数据库表创建成功
  Given 数据库连接正常
  When  执行 alembic upgrade head
  Then  4 张表应创建成功
  And   索引和外键应正确配置

Scenario: ORM 模型可正常操作
  Given 数据库表已创建
  When  通过 SQLAlchemy 创建/查询/更新记录
  Then  所有 CRUD 操作应正常工作
```

## 技术要点

- SQLAlchemy 2.x 使用 Mapped 类型注解
- datetime 字段使用 DateTime(timezone=True)
- JSON 字段使用 SQLAlchemy 的 JSON 类型
- Alembic 管理数据库迁移
